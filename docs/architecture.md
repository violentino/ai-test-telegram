## Архитектура приложения: Telegram бот аналитики рынков (Laravel + Python)

### Цели и возможности
- **Загрузка данных**: Yahoo Finance (через Python `yfinance`) для любых тикеров (BTC-USD, ETH-USD, SOL-USD, XRP-USD и т.д.)
- **Аналитика**:
  - Определение локальных экстремумов цены (max/min)
  - Построение трендовых линий через экстремумы (синие линии)
  - Автоматический поиск уровней поддержки/сопротивления
  - Поиск точек схождения линий (конфлюэнс) и вероятных зон разворота
- **Прогноз/сигналы**:
  - Сигналы о вероятном росте/падении
  - Критические зоны на графике
- **Визуализация**: Экспорт профессионального PNG-графика с разметкой и подписями ключевых уровней/сигналов
- **Монетизация**: Разовый платёж $1 в TON за каждый анализ
- **Интерфейс**: Telegram бот (команды/кнопки), выдаёт изображение + текстовый разбор

### Высокоуровневая схема
- **Telegram Bot (Laravel)**
  - Получает апдейты (webhook)
  - Ведёт диалог с пользователем, валидирует тикер/таймфрейм
  - Управляет платёжным потоком (TON) и проверками оплаты
  - Оркестрирует анализ: ставит задачу в очередь и ждёт результат
  - Отправляет пользователю PNG и описание

- **TON Payments**
  - Интеграция: TON Wallet Pay или публичный TON API (например, `tonapi.io`)
  - Модель оплаты: $1 за один запрос анализа
  - Вебхук подтверждения платежа -> обновление статуса в БД

- **Analysis Service (Python, FastAPI)**
  - Эндпоинт `/analyze` принимает: тикер, период, интервал, опции
  - Загружает свечные данные (Yahoo Finance через `yfinance`)
  - Вычисляет экстремумы (например, `scipy.signal.find_peaks` + фильтрация)
  - Строит трендовые линии (подбор через RANSAC/регрессии/жёсткие связки по экстремумам)
  - Находит уровни S/R (кластеризация `KMeans`/`DBSCAN`, плотности, pivot-based)
  - Ищет точки схождения линий, отмечает критические зоны
  - Генерирует график (Matplotlib) -> PNG (в Base64) + JSON-резюме сигналов

- **Хранилища и инфраструктура**
  - БД: PostgreSQL (или MySQL) для пользователей, платежей, запросов и результатов
  - Кэш/очереди: Redis (очереди для фоновых задач Laravel, кеширование аналитики)
  - Логи и метрики: структурированные логи, при проде — централизованный сбор
  - Объектное хранилище (опционально): S3-совместимый стор для PNG, либо отдача Base64

### Потоки данных (E2E)
1) Пользователь пишет боту тикер (например, `BTC-USD`) и желаемый таймфрейм
2) Бот отвечает ссылкой/кнопкой оплаты TON на $1
3) Пользователь оплачивает; поступает вебхук/поллинг подтверждения
4) Бот ставит задачу «анализ тикера» в очередь (Redis)
5) Воркер Laravel вызывает Python Analysis Service `/analyze`
6) Python сервис возвращает JSON + Base64 PNG
7) Laravel сохраняет результат (БД/кэш), отправляет PNG и текст пользователю

### Основные компоненты
- Laravel (PHP 8.3+)
  - Telegram Bot API (SDK, например `irazasyed/telegram-bot-sdk` или напрямую HTTP)
  - Контроллер вебхука, сервис платёжной логики (TON), сервис аналитики (HTTP-клиент)
  - Очереди: `queue:work`/Horizon, ретраи, тайм-ауты
  - Ограничение частоты (rate limiting), аутентификация вебхуков

- Python FastAPI
  - `yfinance`, `pandas`, `numpy`, `matplotlib`, `scipy`, `scikit-learn`
  - Слой алгоритмов и генерация графики
  - Контракты ввода/вывода (pydantic-схемы), валидаторы

- TON интеграция
  - Способы: Wallet Pay (инвойсы) или мониторинг входящих транзакций на выделенный адрес + комментарий
  - Верификация платежа: сумма, токен, уникальный memo/label, статус «подтверждён»

### Модели данных (упрощённо)
- `users`: telegram_user_id, username, статус
- `payments`: user_id, amount_usd, currency, tx_id, status, created_at
- `analysis_requests`: user_id, ticker, period, interval, status, payment_id
- `analyses`: request_id, summary_json, image_ref (S3/локально/Base64), created_at

### Контракты API
- Laravel -> Python `/analyze` (POST)
  - Вход: `{ ticker, period: "1y"|"6mo"|..., interval: "1d"|"4h"|..., options? }`
  - Выход: `{ image_base64, levels: [...], trendlines: [...], signals: [...], meta: {...} }`

### Надёжность и UX
- Кэширование результатов по (ticker, period, interval) с TTL
- Идемпотентность заданий и платежей
- Параллельная обработка и тайм-ауты; уведомления об ошибках
- Локализация RU/EN в сообщениях бота

### Безопасность
- Секреты в `.env`
- Верификация webhook Telegram/TON
- Ограничение IP для вебхуков (по возможности) и rate limiting

### Развёртывание
- Docker Compose для dev/prod (отдельные профили)
- Обновления без простоя для воркеров/сервиса аналитики
